# Podish Browser on Cloudflare Pages

This directory contains the Cloudflare Pages runtime pieces for serving the
browser app from Pages while loading `rootfs/` objects from R2 through a Pages
Function.

## Layout

- `functions/rootfs/[[path]].ts` serves `GET` / `HEAD /rootfs/*` from the
  `ROOTFS_BUCKET` binding and supports byte ranges for layer blob reads.
- `.dist/` is generated by the deployment script and is safe to delete.

## Deploy

Use the Python deployment script from the repository root:

```bash
python3 Podish.Browser/scripts/deploy_pages.py \
  --project-name your-pages-project \
  --r2-bucket your-rootfs-bucket
```

Optional flags:

- `--publish-dir` to override the .NET publish output.
- `--r2-prefix` to store objects under a different prefix than `rootfs`.
- `--skip-upload` or `--skip-deploy` for partial runs.
- `--branch` for preview deployments.
- `--compatibility-date` if you need to pin a specific Workers compatibility date.

The script stages a Pages-ready static directory, injects `_headers` for
COOP/COEP, uploads `rootfs/` to R2, generates `wrangler.jsonc`, and runs
`wrangler pages deploy`.

Static caching is split by file type:

- Fingerprinted assets under `/assets/*` and hashed runtime payloads under
  `/_framework/*.wasm`, `/_framework/*.wasm.br`,
  `/_framework/dotnet.native*.js`, and `/_framework/dotnet.runtime*.js` are
  served as long-lived immutable assets.
- Stable entrypoints such as `/index.html`, `/_framework/dotnet.js`, and the
  root-level helper modules (`/*.mjs`, `/*.js`) are served with
  `no-cache, must-revalidate` so a new deploy does not strand browsers on an
  old loader that points at removed hashed runtime files.

By default, the deployment script reuses the existing `compatibility_date` from
`wrangler.jsonc`. If the file does not exist yet, it falls back to the current
UTC date so the value does not get rejected as "in the future" when your local
time zone is ahead of UTC.

## Compression

- The deployment script skips precompressed sidecar files such as `*.br` and
  `*.gz`.
- `_framework/*.wasm.br` is kept and deployed. The browser worker rewrites wasm
  boot resource URLs to the Brotli asset path.
- Pages receives canonical static assets and can apply HTTP compression during
  delivery.
- R2 receives canonical `rootfs/` objects only, which keeps byte-range reads
  aligned with the offsets expected by the browser rootfs loader.
