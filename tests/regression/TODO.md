### 2. Understand Instruction Semantics

**Using disassembly tools:**

```bash
# Method 1: Python + capstone
python3 -c "from capstone import *; md = Cs(CS_ARCH_X86, CS_MODE_32); \
print(list(md.disasm(bytes.fromhex('83d0ff'), 0x1000))[0].mnemonic, \
      list(md.disasm(bytes.fromhex('83d0ff'), 0x1000))[0].op_str)"

# Output: adc eax, 0xff
```

**Consult Intel Manual:**
- ADC (Add with Carry): `dest = dest + src + CF`
- Affected flags: CF, OF, SF, ZF, AF, PF

### 3. Design Test Cases

Design test inputs and expected outputs based on instruction semantics:

**Example 1: `ADC EAX, 0xFF` (no carry)**
```python
initial_regs={
    'EAX': 0x100,       # Initial value
    'EFLAGS': 0         # CF=0 (no carry)
},
expected_regs={
    'EAX': 0x1FF        # 0x100 + 0xFF + 0 = 0x1FF
},
expected_eflags=0x0     # Calculate flags based on result
```

**Example 2: `ADC EAX, 0xFF` (with carry)**
```python
initial_regs={
    'EAX': 0xFFFFFF00,
    'EFLAGS': 1         # CF=1
},
expected_regs={
    'EAX': 0x0          # Overflow, result is 0
},
expected_eflags=0x47    # CF=1, ZF=1, PF=1, AF=1
```

## Test Parameters Reference

### Required Parameters

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `initial_regs` | dict | Initial register values | `{'EAX': 0x100, 'ECX': 0x200}` |
| `expected_regs` | dict | Expected register values | `{'EAX': 0x1FF}` |

### Optional Parameters

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `initial_eflags` | int | Initial EFLAGS | `0x1` (CF=1) |
| `expected_eflags` | int | Expected EFLAGS | `0x47` (CF,ZF,PF,AF) |
| `expected_eip` | int | Expected instruction pointer | `0x1003` (after 3-byte instruction) |
| `expected_read` | dict | Expected memory reads | `{0x2000: 0x12345678}` |
| `expected_write` | dict | Expected memory writes | `{0x2000: 0xABCDEF}` |
| `initial_seg_base` | list | Segment bases | `[0,0,0,0,0x3000,0]` (FS=0x3000) |

## Common Instruction Patterns

### 1. Arithmetic Instructions (ADD, SUB, ADC, SBB)

```python
# ADD EAX, 0x10
initial_regs={'EAX': 0x100},
expected_regs={'EAX': 0x110},
expected_eflags=0x0  # No flags set
```

### 2. Logical Instructions (AND, OR, XOR)

```python
# XOR EAX, EAX (common zero pattern)
initial_regs={'EAX': 0x12345678},
expected_regs={'EAX': 0x0},
expected_eflags=0x44  # ZF=1, PF=1
```

### 3. Shift Instructions (SHL, SHR, SAR)

```python
# SHL EAX, 2 (left shift 2 bits)
initial_regs={'EAX': 0x1},
expected_regs={'EAX': 0x4},
expected_eflags=0x0
```

### 4. Stack Operations (PUSH, POP)

```python
# PUSH EAX
initial_regs={'EAX': 0x12345678, 'ESP': 0x8000},
expected_regs={'ESP': 0x7FFC},  # ESP -= 4
expected_write={0x7FFC: 0x12345678}
```

### 5. Memory Operations (MOV m32, r32)

```python
# MOV [0x2000], EAX
initial_regs={'EAX': 0xDEADBEEF},
expected_regs={'EAX': 0xDEADBEEF},  # EAX unchanged
expected_write={0x2000: 0xDEADBEEF}
```

### 6. Conditional Jumps (JE, JNE, JG, JL)

```python
# JE offset (conditional jump when ZF=1)
initial_eflags=0x40,  # ZF=1
expected_eip=0x1008   # Jump to target address
```

### 7. Compare Instructions (CMP, TEST)

```python
# CMP EAX, 0x100 (EAX == 0x100)
initial_regs={'EAX': 0x100},
expected_regs={'EAX': 0x100},  # CMP doesn't modify registers
expected_eflags=0x44  # ZF=1, PF=1
```

### Example 1: MOV Instruction

```python
@pytest.mark.regression
def test_id_4_mov_r32_r32():
    """Test: ID_4: mov ebx, eax"""
    runner = Runner()
    # Raw: 89c3
    # Instruction: MOV EBX, EAX
    # Semantics: EBX = EAX
    assert runner.run_test_bytes(
        name='ID_4: mov ebx, eax',
        code=binascii.unhexlify('89c3'),
        initial_regs={
            'EAX': 0x12345678,
            'EBX': 0xFFFFFFFF  # Initial value will be overwritten
        },
        expected_regs={
            'EBX': 0x12345678  # EBX now equals EAX
        }
        # MOV doesn't affect flags, so no expected_eflags needed
    )
```

## FAQ

**Q: How do I know which flags an instruction modifies?**  
A: Check Intel manual or https://www.felixcloutier.com/x86/, each instruction page has "Flags Affected" section.

**Q: Test fails with "Opcode Not Implemented", what to do?**  
A: This instruction is not yet implemented in `src/ops.cpp`. Skip this test or submit an issue requesting implementation.

**Q: How to calculate correct EFLAGS value?**  
A: Refer to existing tests or use Unicorn/real CPU to run instruction and observe result. Flag bit definitions:
- CF (bit 0): Carry
- PF (bit 2): Parity
- AF (bit 4): Auxiliary carry
- ZF (bit 6): Zero
- SF (bit 7): Sign
- OF (bit 11): Overflow

**Q: How to choose memory addresses?**  
A: Use `0x2000` (data region) or stack addresses. Framework automatically maps `0x1000-0x2000` (code), `0x2000-0x3000` (data), `0x7000-0x9000` (stack).

**Q: Can I use Unicorn to auto-generate expected values?**  
A: Yes! Don't fill `expected_regs`, run test to see Unicorn's results, then manually verify and fill in.
